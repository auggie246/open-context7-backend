# Open Context7 Backend Development

Purpose: explain the local development workflow for Open Context7 Backend.

Source of truth: `AGENTS.md`, `pyproject.toml`, `app/`, `tests/`, `scripts/verify_code_quality.sh`, and the existing `docs/` set.

## Prerequisites

- Python 3.12 or newer.
- `uv` for dependency management and command execution.
- Docker only when testing Qdrant or the Compose stack.

Install dependencies:

```sh
uv sync
```

The package entrypoint is:

```sh
uv run context7-backend --help
```

## Project Layout

- `app/`: FastAPI service, CLI, catalog loading, ingestion, retrieval, formatting, auth, and settings.
- `tests/`: pytest coverage for API behavior, parsing, ingestion, catalog validation, settings/auth, retrieval, docs contract, and scaffold behavior.
- `libraries/*.yaml`: catalog metadata source of truth for library IDs, versions, aliases, source paths, and include/exclude globs.
- `examples/`: local example documentation used by ingest and QA commands.
- `docs/`: human documentation for the local backend.
- `scripts/`: quality and contract QA scripts.
- `.local-store/chunks.json`: local generated chunk store used by lexical retrieval.
- `src/context7--upstash-context7-mcp-3.2.1`: vendored reference material. Do not edit, format, lint, or commit changes under this tree.

## Local Run Loop

Ingest example docs:

```sh
uv run context7-backend ingest --library /internal/platform --version main --source-dir examples/platform-docs
```

Start the API:

```sh
uv run uvicorn app.main:app --host 127.0.0.1 --port 8000
```

Check the server:

```sh
curl -f http://127.0.0.1:8000/healthz
curl 'http://127.0.0.1:8000/api/v2/context?libraryId=/internal/platform&query=valuesFrom'
```

## Quality Commands

Run the fast test suite:

```sh
PYTHONDONTWRITEBYTECODE=1 uv run pytest -q -p no:cacheprovider
```

Run lint and type checks:

```sh
uv run ruff check app tests
uv run basedpyright app tests
```

Run the repository quality gate:

```sh
scripts/verify_code_quality.sh
```

That script removes bytecode artifacts under `app` and `tests`, runs Ruff, BasedPyright, and pytest, then checks that no `__pycache__` or `.pyc` files remain under those directories.

## Tests

Tests live in `tests/`. Focused examples:

```sh
PYTHONDONTWRITEBYTECODE=1 uv run pytest -q -p no:cacheprovider tests/test_docs_contract.py
PYTHONDONTWRITEBYTECODE=1 uv run pytest -q -p no:cacheprovider tests/test_ingest_cli.py
PYTHONDONTWRITEBYTECODE=1 uv run pytest -q -p no:cacheprovider tests/test_settings_auth.py
```

Use targeted tests while editing a narrow surface, then run the quality gate before handing off changes that affect app behavior.

## Development Constraints

- Keep ingestion deterministic. Do not add LLM extraction to parsing or indexing.
- Keep `libraries/*.yaml` as the catalog metadata source of truth.
- Keep `/healthz` unauthenticated.
- Keep `/api/v2/*` protected by bearer auth when `DOCS_API_KEYS` is configured.
- Preserve constant-time API key comparison through `secrets.compare_digest`.
- Do not edit vendored source under `src/context7--upstash-context7-mcp-3.2.1`.
- Avoid generated bytecode artifacts in `app` and `tests`.

## Gotchas

- `.local-store/chunks.json` is generated local retrieval state, not catalog metadata.
- The vendored Context7 source is reference material only.
- If QA starts services, clean them up and check ports `8000` and `6333`.
